# Bug Fixes - December 9, 2025

## Overview
This document covers two bug fixes implemented on December 9, 2025:
1. Project published date not updating in ContentTimeline
2. Page scroll position not resetting when navigating between routes

---

## Bug Fix 1: Project Published Date Not Updating in Timeline

### Problem
When updating a project's `publishedAt` field, the ContentTimeline table was not being updated with the new date. This caused the "Latest Activity" section on the homepage to show incorrect dates.

### Root Cause Analysis
The issue was in `backend/src/services/project.service.ts`:

1. **In `createProject` function (line 245):**
   - Timeline was using `date: newProject.createdAt`
   - Should use `publishedAt` if provided, fallback to `createdAt`

2. **In `updateProject` function (line 366-376):**
   - Timeline update only modified `title`, `excerpt`, and `url`
   - The `date` field was completely missing from the update

### Solution
Updated both create and update functions to use `publishedAt || createdAt`:

**Create Function (line 245):**
```typescript
await tx.contentTimeline.create({
  data: {
    contentType: 'PROJECT',
    contentId: newProject.id,
    title: newProject.title,
    excerpt: newProject.overview,
    date: newProject.publishedAt || newProject.createdAt, // Fixed
    url: `/projects/${newProject.slug}`,
    tags: tags.map((t) => t.name),
  },
});
```

**Update Function (lines 366-377):**
```typescript
// Update timeline if exists
await tx.contentTimeline.updateMany({
  where: {
    contentType: 'PROJECT',
    contentId: id,
  },
  data: {
    title: updated.title,
    excerpt: updated.overview,
    url: `/projects/${updated.slug}`,
    date: updated.publishedAt || updated.createdAt, // Added
  },
});
```

## Bug Fix 2: Page Scroll Not Resetting on Navigation

### Problem
When clicking links from the homepage to navigate to different pages (Projects, Publications, Guidebooks), the browser would maintain the previous scroll position instead of scrolling to the top of the new page.

### Root Cause Analysis
React Router does not automatically scroll to the top when navigating between routes. This is by design to support features like scroll restoration, but it requires explicit handling for the default "scroll to top" behavior.

### Solution
Created a `ScrollToTop` component that:
1. Uses React Router's `useLocation` hook to detect route changes
2. Scrolls window to position (0, 0) whenever the pathname changes
3. Is placed inside `BrowserRouter` to access routing context

**New Component:**
```typescript
// frontend/src/components/ScrollToTop.tsx
import { useEffect } from 'react';
import { useLocation } from 'react-router-dom';

export function ScrollToTop() {
  const { pathname } = useLocation();

  useEffect(() => {
    window.scrollTo(0, 0);
  }, [pathname]);

  return null;
}
```

**Integration in App.tsx:**
```typescript
<BrowserRouter>
  <ScrollToTop />
  <Routes>
    {/* ... routes ... */}
  </Routes>
</BrowserRouter>
```
